Mysql基础

发表于 2018-01-07 | 分类于 数据存储
Mysql基本操作-用户管理，库表操作以及常用函数

Mysql

  • 关系型数据库(Relational Database Management System)
  • 层级关系：
        库(schema)：存放数据表
        表(table)：存放数据
        字段(field)：数据表头
            列(column)、行(row)

  • 查看命令帮助：help SQLCOMMAND
  • 命令结束符号：;

用户管理


存放Mysql用户信息的表：mysql.user

创建用户

  • mysql> CREATE USER ‘USER’@’HOST’ IDENTIFIED BY ‘PWD’;
        自带创建用户语句
        自动调用password()函数对密码进行MD5加密
  • mysql> grant all privileges on DB.TABLE to ‘USER’@’HOST’ identified by ‘PWD’
        授权语句
  • mysql> inset into mysql.user(Host, User, Password) values(‘localhost’, ‘inserin’, password(‘123456’))
        插入数据语句

设置密码

  • $ mysqladmin -u USER password NEWPWD
  • mysql> set password for ‘USER’@’HOST’ = password(NEWPWD)
  • mysql> update mysql.user set Password=PWD where User=USER
        不指定where将所有Password字段都修改

删除用户

  • mysql> drop user ‘USER’@’HOST’

库表操作

库管理

  • mysql> show databases;：查看当前可以操作的所有库
  • mysql> create database DBNAME;：创建一个新的数据库
  • mysql> drop database DBNAME;：删除一个可操作的库
  • mysql> use DBNAME;：使用一个可操作的库

表管理

  • [DBNAME]> show tables;：查看当前库下的所有可操作的表
  • [DBNAME]> create table TBNAME(field type attr, …)：创建一张新表
                  TYPE
整数
int(m)：4个字节 范围(-2147483648~2147483647)
小数
float(m,d)：单精度 8位精度(4字节) 
double(m,d)：双精度 16位精度(8)字节
         m-总个数、d-小数位
字符串
char(n)：固定长度，最多255个字符，字符串末尾不能有空格
     若存入字符数小于n，则以空格补于其后，查询之时再将空格去掉。检索速度比varchar快
varchar(n)：固定长度，最多65535个字符
text：可变长度，最多65535个字符
日期
date: 'year-month-day'
time: 'hour-minute-second'
datetime: 'year-month-day hour-minute-second'
                ATTR - 属性
NULL：可以为空
NOT NULL：不可以为空
DEFAULT：默认值
PRIMARY KEY：主键
AUTO_INCREMENT：自动递增(整数类型)
UNSIGEND：无符号
  • [DBNAME]> alter table TBNAME [options];：修改表字段
        options
add：增加字段
    alter table TBNAME add FIELD ATTER
change：修改字段名称
    alter table TBNAME change OLDFIELD NEWFIELD ATTR
modify:修改字段类型
    alter table TBNAME modify FIELD NEWATTR
drop：删除某个字段
    alter table TBNAME drop FIELD
  • [DBNAME]> desc TBNAME;：查看该表的表结构(字段信息)
  • [DBNAME]> insert into TBNAME(field,…) values(v1,…),(v2,…),…;：向表中写入数据
  • [DBNAME]> update TBNAME set FIELD=VALUE where FILED=VALUE;：修改表中对应where FIELD=VALUE的行
        不使用where精确匹配行将会修改所有行
  • [DBNAME]> delete from TBNAME where FIELD=VALUE;：删除字段相对应的行
        不使用where精确匹配行将会删除所有数据
        用于删除表中的数据
  • [DBNAME]> drop table TBNAME;：删除一张表

数据查询

select *[filed] from TBNAME [where condition]

子语句

  • where：精确查询
  • like：模糊查询
        % - 0或多
        _ - 任意一个
        [] - 列表内任意一个
  • 比较
    select * from TBNAME where number >|<|= NUMBER;
        >|>=
        <|<=
         =
        in(v1,v2,…)
  • 逻辑判断
    select * from TBNAME where filed=value and | or field=value
    [and 1=1 | or 1=1]
        and
        or
        not
  • 子组结果模式: ()
    select * from TBNAME where FIELD=(select filed from TBNAME where filed=value)

表联合

  • select * from TB1,TB2 where TB1.field = TB2.field
        必需两表存在相同的字段值
        相对应的字段再同一行输出
        如果匹配字段值不相同则不会输出
  • select * from TB1 inner join TB2 on TB1.field = TB2.field
        效果同上
  • select * from TB1 left outer join TB2 on [condition]
        TB1在TB2左边
        condition必需成立
        如果多出的行没匹配到TB2，用NULL填充
  • select * from TB1 right outer join TB2 on [condition]
        TB1在TB2右边

常用函数

  • union：联合查询
    sqlcommand1 union sqlcommand2
        两条数据库语句一起执行输出
        select输出的字段数必须相同

  • order by FIELD [desc]：对字段进行排序
        默认升序
        desc - 降序
        用于判断输出的字段数
            order by NUMBER(对应字段)，超出输出的字段值便会抛出SQL语句错误，相当与超出索引

  • group by FIELD：对该字段里相同值的行进行分组
    一般配合统计函数使用：
        max(field)：输出该组的最大值
        min(field)：输出该组的最小值
        sum(field)：输出该组的数值总和
        avg(field)：输出该组的平均值
        count(field)：该组的总数(行数)

  • limit [offsize,]N：限制输出行数
        offsize：行索引(0开始)
        N：控制输出的行数
        limit N：输出N行 -> limit 0,N
            从索引为0的行输出
        limit 0,1：只输出第一行
        limit 1,1：只输出第二行
  • 字符操作
    ascii(str)：将字符串转换为ascii码
        只能转单个字符
    left(a,b)：字符截取输出，截取a的前b个字符
    substr(a,b,c)：字符截取输出，在a字符串中，从b位置开始截取c长度个字符
