OSI七层模型

发表于 2018-01-04 | 分类于 计算机网络基础
计算机网络通信基本架构

OSI - Open System Interconnect
    有国际化标准组织(ISO)和国际电报电话咨询委员会(CCITT)联合定制的开放系统互连参考模型
    目的：为计算机互连提供一个共同的基础和标准框架，并为保持相关标准的一致性和兼容性提供共同参考

大概模型图(应表会传网链物)

物理层(Physcical Layer)

建立在传输媒介的基础上，实现设备之间的物理接口建立、维护、取消物理连接作用

单位：比特流
设备：光纤、同轴电缆、双绞线(网线)、中继器、集线器

  • 带宽限制 - 每秒能通过的比特数，减少信道拥堵
        点到点
        一点到多点
        串行、并行
        半双工、全双工
        同步、异步

数据链路层(Data Link Layer)

使用系统的硬件地址或物理地址(MAC)来寻址、数据成帧、数据检错、重发

单位：数据帧 + MAC地址
设备：二层交换机、网桥、网卡
协议：PPP(PPTP\L2TP)、IEEE802.x、STP、帧中继
        PPP(PPTP\L2TP)：点对点传输协议 - 一般用于VPN(Virtual Private Network-虚拟专用网络)
        STP：交换机的生成树协议

网络层(Network Layer)

路由寻址，实现不同网络之间的路径选择

单位：数据包(分组) + IP地址
设备：网关、路由器
协议：ARP\RARP、IP、ICMP、RIP、OSPF[EIGRP|IGRP]
        ARP：地址转换协议 - 实现物理地址寻址(IP <=> MAC)
        IP：网际协议 - 路由寻址
        ICMP：网际控制报文协议 - Ping
        RIP：路由信息协议 - 用于路由器，基于距离矢量，以跳数作为计数标准(MAX:15)
        OSPF[EIGRP|IGRP]：开放式最短路径优先协议， EIGRP|EGRP - Cisco独有

传输层(Transport Layer)

定义数据传输的协议端口号，以及流控和差错校验

单位：数据段 + 端口
端口(PORT)：
    数量：65535(1~1023为系统使用端口)
    功能：网络基于Socket(套接字)之间的通信
    常用(netstat)：8080 - www、20[21] - ftp、22 - ssh、23 - telnet、25 - SMTP、80 - www、110 - POP3、161 - SNMP、137[138\139] - NETBIOS、3306 - Mysql
协议：TCP/UDP
    TCP：面向连接,可靠的传输协议,三次握手
    UDP：面向无连接，高效的传输协议
    透明传输，不代表安全

会话层(Session Layer)

建立、管理、终止会话

单位：数据报文

  • 对应主机进程，本地主机与远程主机正在进行的会话

表示层(Presentation Layer)

数据的表示、编码格式转换、安全|加解密、压缩

单位：数据报文
格式：JPEG、ASCII、DECOCO、加密格式…

应用层(Application Layer)

用户与网络服务的接口

单位：数据报文
功能：提供网络管理、文件传输、数据处理…
协议：Telnet、FTP、HTTP[S]、SNMP、DNS…
