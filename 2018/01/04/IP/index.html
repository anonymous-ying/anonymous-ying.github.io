IP

发表于 2018-01-04 | 分类于 计算机网络基础
IP协议分析，IP地址\子网掩码划分

IP(Internet Protocol)：网际协议，实现计算机网络连接之间进行通信

数据包

  • 面向无连接，以分组的形式进行传输，可以通过不同路径到达目的
  • 每一个包(分组)，都作为一个独立的报文传送
  • 数据包长度：30000字节(Bytes), 1字节 = 8位二进制
    数据包格式


    版本：ipv4(0x4), ipv6
    首部长度：用于确定数据报中的偏移位置
    总长度：结合首部长度字段，就可以知道IP数据中数据内容的起始位置和长度
    标识、标志、片偏移：用于数据的分片重组
    生存时间：数据报文可以经过的最大路由跳数，由TTL(32/64)控制，经过一个路由-1，当该字段为0时数据报被丢弃，并发送ICMP通知源主机
    头校验和：检错，确认封包被正确无误的接收到

          分片/重组
原因：数据链路层具有最大传输单元(MTU)特性，限制了数据帧的最大长度，如过数据数据包长度超过了MTU，数据包就要进行分片

  • 源IP：负责分片
  • 目的IP：负责校验重组
  • 透明传输
  • 分片过的数据包可能不止一次的再进行分片，避免丢包

IP地址

32位的二进制数组成，采用点分的十进制表示
11111111.11111111.11111111.11111111 <=> 255.255.255.255

分类

标识码(ID)：网络ID(由子网掩码划分)、主机ID
    同个网络断使用同一个网络ID
    每台独立的主机使用一个主机ID

  • A类：第一个字节(8位二进制)，为网络ID
    IP地址范围：1.0.0.0 ～ 126.255.255.255
    默认子网掩码：255.0.0.0
    私有地址：10.0.0.0 ～ 10.255.255.255
    每个网段容纳主机数：256^3-2
  • B类：第一第二字节为网络ID
    IP地址范围：128.0.0.0 ~ 191.255.255.255
    默认子网掩码：255.255.0.0
    私有地址：172.16.0.0 ～ 172.31.255.255
    每个网段容纳主机数：256^2-2
  • C类：第一第二第三字节为网络ID
    IP地址范围：192.0.0.0 ~ 223.255.255.255
    默认子网掩码：255.255.255.0
    私有地址：192.168.0.0 ～ 192.168.255.255
    每个网段容纳主机数：256-2
  • D类：不分网络ID、主机ID，用于组播
    IP地址范围：224.0.0.0 ~ 239.255.255.255
  • E类：不分网络ID、主机ID，网络保留段
    IP地址范围：240.0.0.0 ~ 255.255.255.254
  • 特殊地址
    169.254.x.x：DHCP自动获取IP错误返回地址
    源IP地址每个网段的第一个IP地址，表示网段本身。0.0.0.0表示当前主机
    广播地址：每个网段的最后一个IP(255)，向所有可通信的主机发送数据，容易造成网络负担，一般路由器都阻止该报文
    环回地址127.0.0.0 ~ 127.255.255.255，主要用来测试主机是否能正常工作(127.0.0.1)

子网掩码(netmask)


用于划分网络ID\网段，对应四个字节(32位)
一般：255.0.0.0/8 | 255.255.0.0/16 | 255.255.255.0/24 | 255.255.255.255/单台主机
一个字节(二进制)：11111111
2^7 + 2^6 + 2^5 + 2^4 + 2^3 + 2^2 + 2^1 + 2^0 <=> 128+64+32+16+8+4+2+1 = 255

子网划分(B类开始)


255.255.255.0 -> 容纳254台：256-2(0~255 = 256, 排除第一个最后一个-2)，C类
    (192.168.1.1/24 0-255)
255.255.255.128 -> 容纳126台：256-128-2
    (192.168.1.1/25 0-128)
255.255.255.192(128+64) -> 容纳62台：256-192-2
    (192.168.1.1/26 0-64)
255.255.255.224(128+64+32) -> 容纳30台：256-224-2
    (192.168.1.1/27 0-32)
255.255.255.240(128+64+32+16) -> 容纳14台：256-240-2
    (192.168.1.1/28 0-16)
255.255.255.248(128+64+32+16+8) -> 容纳6台：256-248-2
    (192.168.1.1/29 0-8)
255.255.255.252(128+64+32+16+8+4) -> 容纳2台：256-252-2
    (192.168.1.1/30 0-4)
