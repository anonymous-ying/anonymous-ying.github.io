关系与非关系型数据库

  • 关系型(Oracle\SQLServer\Access\MySql) - 可以进行复杂的表操作，支持事物(ACID)

    事物：建表->写入数据，或者是一条SQL语句的执行的抽象说法
ACID：
Atomicity(原子性)：要么做，要么不做
Consistency(一致性)：事物开始之前和事物结束以后，数据库的完整性约束没有被破坏
Isolation(隔离性)：要么只读，要么只写(脏读、不可重复读、幻读、丢失更新)
                 脏读：一个事物读取了另一个事物未提交的数据，这个数据有可能回滚
                 不可重复读：两个相同的查询返回不同的结果，由于查询时系统中其他事物修改的提交引起
                 幻读：两个事物操作同时进行，导致出现幻觉一样
                 丢失更新：两个事物同时修改一条数据，造成数据覆盖
Durability(持久性)：一旦事物提交成功，数据修改是永存的

        数据存储于硬盘中，需要I/O开销
            不利于大规模读写 => 需要做主从复制来实现读写分离或者库站分离，来提高性能 => 不适用：社交、搜索类型的平台
        存储数据的格式：只支持基础类型，一般都有固定的表结构，通过外键关联来建立表与表之间的关系

  • 非关系型(memcached\Redis\Mongodb\HBase) - 减少不必要功能来提升性能
        数据存储于缓存中，查询速度快
            适用于高并的读写
        存储格式：支持多变存储机制(键值，文档，图形)，数据没有耦合性，容易扩展

            键值存储保存方式
临时性：memcached
    在内存中保存数据
    可以快速进行读写
    数据可能丢失
永久性(类似关系型)
两者兼备(临时+永久):Redis
    首先把数据存在内存中，满足特定时间后写入到硬盘中
=========================
        面向文档(mongodb)
无需定义表结构
可以使用复杂的查询条件
  • 不足:
        关系型：读写差、固定表结构
        非关系型：弱化了存储的操作的功能

mysql


关系型数据库，python模块：MySQLdb，注意模块名大小写

  • 阅读Mysql基础操作，掌握SQL语句忽略
  • 创建Mysql-API对象
    conn = MySQLdb.connect(host=’HOST’, user=’USER’, passwd=’PWD’[,db=’USEDB’])
        USEDB必需事先存在，可以为空
        返回值：connect对象
  • 创建游标(类似MysqlShell)
    cur = conn.cursor()
  • 执行SQL语句
    cur.execute(‘SQL’)：一次执行一条SQL语句
        SQL语句不用分号结束
    cur.executemany(‘SQL’,LIST(VALUETUPLE))：一次执行多条语句
        参数二：元组形式的列表，元组为SQL对应的VALUE
    cur.commit()：当执行多条语句，或者执行一个事物的时候使用，进行数据库刷新
  • cur.fechall()：接收语句的查询结果(SQL要有返回信息)
        返回值类型：多重元组
  • 关闭游标，关闭连接对象：cur.close(),conn.colse()

示例

mongodb


非关系型数据库，python模块：pymongo

  • Linux下安装开启服务
        安装：sudo apt-get install mongodb
        开启：sudo service mongod start
        查看：cat /var/log/mongodb/mongod.log | sudo service mongod status
  • 创建MongoDB-API对象
    client = pymongo.MongoClient(‘HOST’, PORT)
        默认端口：27017
  • 创建/使用层级关系(库->表)
        库创建：db = client[‘DBNAME’]
        创建表：tb = db[‘TBNAME’]
  • 写入数据
        数据格式：字典
    post_data = {
    'title': 'Python and MongoDB',
    'content': 'PyMongo is fun, you guys',
    'author': 'wyy'
}
    tb.insert_one(dictDATA)：一次写入一条
            一般配合for循环写入数据集
    for index, line in enumerate(lines):
    data = {
        'index':index,
        'line':line,
        'words':len(line.split())
    }
    #往表单中拆入数据，字典的形式
    sheet_lines.insert_one(data)
    tb.insert_many([dictDATA,…])：一次写入多条
    tb.update({key:value}, {‘$set’:{key:newvalue}})：修改一条数据
    tb.remove({key:value})：删除一条数据
    tb.delete_many({‘$key’:{‘CONDITION’:value,..}})
  • 读取数据
    td.find({key,value}, [field1, field2][,skip=1, limit=10])
        value => {$*, value}：对value进行条件判断输出
                $lt : <
                $lte : <=
                $gt : >
                $gte : >=
                $regex：支持正则表达式
                $exists：是否存在该键
    返回值：类对象，需要遍历输出
    BIF：
        find().count：统计数目
        find().sort：排序

csv


excel格式存储

  • 写入数据
        1、写/追加模式打开一个存数据的文件
            with open(file.csv, ‘w|a’) as f
            详细解析看Python文件操作章
        2、创建csv句柄(写入csv的对象)
            fw = csv.write(f[,delimiter=’\t’,lineterminator=’\n\n’])
            参数一：打开的文件对象
            参数二：设置数据分隔符，默认逗号
            参数三：设置行分隔符
        3、写入数据
            fw.writerow(LIST)：列表的格式写入数据
  • 读取数据
        1、读模式打开csv文件对象
        2、创建csv读句柄
            rw = csv.reader(f)
        3、将rw转换为列表
            data = list(rw)

示例

import csv

csvfile = 'csvData/test.csv'
'''写'''
# 打开文件
with open(csvfile, 'w') as f:
    # 实例化csv的写方法
    fw = csv.writer(f)
    for i in range(1, 100):
        # 写入每行
        fw.writerow([i, i + 1, i * 2, i ** 2])

print "write success!!!"

'''读'''
with open(csvfile, 'r') as f:
    fr = csv.reader(f)
    fdata = list(fr)
    print fdata

print "---------------------------"
'''文件读取'''
with open(csvfile, 'r') as f:
    for i in f.readlines():
        print i

序列化数据存储

  • 序列化：把内存数据转为字符串，反之为反序列化
  • 一般文件只能写入字符或二进制，解决文件不能写入列表、字典类型
    TypeError: expected a string or other character buffer object

json


所有语言通用的序列化存储方式，仅支持基本的序列化结构(列表、字典…)

  • 写入内存数据
        1、写/追加模式打开文件对象
    with open(‘data.txt’, ‘w|a’) as f
        2、调用json序列化数据写入
    f.write(json.dumps(DICT))
            数据不一定是字典格式，可以是其他类型的内存数据
            dumps需要手动调用write方法
    json.dump(DICT,fp)
            dump不需要调用write
  • 读取内存数据
        1、读模式打开文件对象
        2、调用json反序列化字符串成内存数据
    json.loads(f.read())
            loads需要手动调用read方法
    json.load(fp)
            load不需要调用read方法

shelve


python独有的序列化数据存储方式，可以支持类对象函数

  • 进一步封装并优化了pickle模块(类似json模块，只支持python)
  • 以键值的方式进行序列化存储
  • 如果为存储类对象函数，只能封装其对应的内存指针，而且必需在类函数存在的前提下才能运行
  • 创建shelve句柄
    she = shelve.open(file)
        读写都是使用该方法
        无需先打开文件句柄
  • 读写数据
        直接使用键值赋值方式即可写入，写入后close文件就会将数据序列化存储
            she[key]=value
                key：在读取时同过key来调用序列化的数据
  • BIF
    she.dict：字典形式返回存储数据，如果不存在函数类不会报错
    she.items()：列表的形式返回数据，需要加载函数类，如果函数类不存在会报错
    she.get(KEY) = she[KEY]：获取一个键对应的值
    she.close()：关闭shelve句柄

示例
